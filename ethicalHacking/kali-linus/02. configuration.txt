Below configurations are for instance when Kali linux is installed on virtual machine

1. `ping google.com` : To check the internet connectivity for virtual machine
    - If it displays ping results, then internet is working
    - If there is no results then go to devices -> network -> network settings, on virtual machine options
    - Select 'Bridge adapter' in 'attached to' field
    - Select the suitable addapter in 'name' field
    - To know the adapter for you system, run ifconfig on terminal
    - Select 'Allow VMx' for promiscuous mode
    - Check again for the internet connectivity
    - If there is no internet connection, then go to /etc/NetworkManager
    - In the folder, edit NetworkManager.conf and modify 'managed' to true is not already.
    - Once done, run `service network-manager restart`

2. Run `apt-get update` to update the repository list
3. Run `apt-get upgrade` to upgrade all the installed packages
4. Add kali repository
    - echo "deb http://http.kali.org/kali kali-rolling main non-free contrib" | sudo tee /etc/apt/sources.list
    - echo "deb http://http.kali.org/kali kali-last-snapshot main non-free contrib" | sudo tee /etc/apt/sources.list
    - `apt-get update`
5. Install guest addition (Steps from https://www.kali.org/docs/virtualization/install-virtualbox-guest-additions-kali/)
    - For new version of kali
        - `sudo apt-get update`
        - `sudo apt-get install -y virtualbox-guest-x11`
        - `reboot`
    - For older version of kali
        - `apt-get install -y dkms linux-headers-$(uname -r)
        - Select 'devices -> Install Guest Addition`. Click on cancel on error message
        - go to `/media/cdrom0`, copy  VboxLinuxAdditions.run to home directory
        - Run `./VboxLinuxAdditions.run` from home folder
        - `reboot`
